다음 문제는 어떻게 풀까?
수열 (2,2,7,7,7,7,5,7,7)이 주어졌을 때, 연속해서 나오는 같은 숫자를 한 묶음이라 봤을때, 총 몇개의 묶음이 있을까?

눈으로 보면 (2,2), (7,7,7,7), (5), (7,7) 이렇게 4개의 묶음이 있다는 것을 알 수 있다.
이 과정을 코드로는 어떻게 구현할 수 있을까?

그렇다면 그 순간을 잡기 위한 조건은 어떻게 될까?
단순히 생각하면, 새로운 묶음이라는 것은 바로 직전에 나온 숫자와 다른 숫자가 나왔다는 의미를 갖기 때문애,
i번째 index에 해당하는 원소를 a[i]라고 했을 때, a[i] != a[i-1]인 경우를 찾으면 된다.
 
단, i가 0인 경우에는 직전 원소를 정의할 수 없으므로, 따로 예외적으로 처리를 해줘야 함에 유의한다.
즉, 다음 조건을 만족하는 경우에 개수를 증가 시키면 된다.
- i가 0이거나 a[i] != a[i-1]인 경우
코드로 구현해 보면 다음과 같다.

cnt = 0
for i in range(n):
    if i == 0 or a[i] != a[i-1]
        cnt += 1
print(cnt)

//Side Note

만약 다음과 같이 코드를 구현하면 결과가 어떻게 나올까?

cnt = 0
for i in range(n):
    if a[i] != a[i-1] or i == 0:
        cnt += 1
print(cnt)

Runtime Error가 발생한다. 그 이유는 i가 0인 경우이 a[i] != a[i-1]에 대한 비교를 진행하기 때문이다.
그렇다면 if문이 다음과 같을 때에는 왜 Runtime Error가 발생하지 않는가?
if i == 0 or a[i] != a[i-1]:

if문의 경우 앞에 있는 조건을 먼저 확인한다.
or 연산자의 경우에는 둘 중 하나라도 참이면 되기 때문에, 앞에 i == 0 조건이 참인 경우에는 뒤의 조건들을 전혀 확인하지
않고 바로 참이라고 판단을 하고 넘어가게 된다.
따라서 a[i] != a[i-1] 조건은 i가 0 이 아닌경우에만 보게 되기 때문에 Runtime Error가 발생하지 않게 된다.
즉, 범위의 대한 조건은 꼭 if 조건문의 가장 앞에 써줘야만 한다.
